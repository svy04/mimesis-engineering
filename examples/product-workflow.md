# Example: Product Workflow Mimesis

Use this when a product workflow exists but does not yet show its input, transformation, proof, and next action clearly.

This scenario is informed by the `mimesis-canvas` product example.

## Inputs

- current workflow or product page
- trusted docs or onboarding examples
- public changelog examples
- one case study structure
- current proof boundary

## Output

- who it is for
- input path
- transformation path
- deliverables
- proof boundary
- next proof plan
- status and limitations

## Seven-Day Version

Ship one audit or workflow page with:

- input
- analysis
- claim boundary
- next proof plan

## Boundary

This example does not prove product value.
It gives a structure for making the workflow inspectable.
