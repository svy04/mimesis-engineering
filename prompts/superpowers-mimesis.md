# Superpowers + Mimesis Prompt

Use Superpowers for process discipline.
Use Mimesis Engineering for artifact standards.

Give AI standards, not roles.
Bring one weak artifact.

Import:
Read the current repo/runtime state, relevant instructions, and the weak artifact before proposing changes.

Distill:
State the goal, exclusions, proof boundary, and the smallest useful slice.

Capsule:
Turn the slice into one verifiable unit with a failing audit, checklist, or test when implementation changes behavior.

Shard:
Edit the smallest files needed. Preserve the local style and the `.mimesis` artifact trail.

Verify:
Run the focused audit first, then broader preflight only when the focused audit passes.

Remember:
Update `.mimesis/run_ledger.md` with evidence-backed results and remaining open gates.

Do not invent proof.
Do not claim external adoption, benchmarked productivity, package publication, shipped plugins, legal originality, or real user outcomes unless direct evidence exists.
