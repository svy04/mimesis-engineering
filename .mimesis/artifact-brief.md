# Artifact Brief

## Artifact

`svy04/mimesis-engineering` public repository surface.

## Audience

AI-native builders, repo visitors, contributors, and operators who want a repeatable artifact-first AI workflow.

## Goal

Make a stranger understand Mimesis in 30 seconds and let a builder run a first loop in 5 minutes.

## Current Weakness

The core idea is strong, but it can look like a prompt philosophy unless the repository exposes concrete files, templates, reference packs, cases, and proof boundaries.

## Constraints

- no fake engagement
- no hidden references
- no surface copying
- no unsupported productivity or adoption claims
- no claim that idea-stage plugins are shipped
- keep the first version usable with local Markdown files

## Success Looks Like

The repository reads as an AI-native work framework with a visible protocol:

- README explains the hook and first loop quickly.
- `spec/` defines the framework contract.
- `.mimesis/` records the artifact trail.
- templates and reference packs let users try it.
- cases show before/after proof boundaries.
- adapters/plugins have status labels and do not overclaim implementation.

## Proof Needed

File-level evidence, local link checks, issue form checks, whitespace checks, and a run ledger.
