# Transformation Plan

## Starting Artifact

The repository already had a strong concept and several docs, but the v0.1 public framework surface was incomplete.

## Extracted Patterns To Use

- hook first
- first action early
- concrete output artifacts
- explicit protocol files
- contribution paths
- proof and originality boundary near the public surface
- integration status labels

## Patterns To Transform

- Convert philosophy into copyable files.
- Convert "prompt pack" perception into framework protocol.
- Convert future integrations into clearly labeled contracts and ideas.
- Convert case claims into before/after and remaining-gap language.

## Patterns To Reject

- fake engagement loops
- hidden reference borrowing
- overclaiming adoption
- claiming plugin implementation before it exists
- treating prompts as the product

## New Structure

The repository surface should expose:

- `README.md` for 30-second understanding and 5-minute action
- `spec/` for framework contracts
- `.mimesis/` for the current artifact trail
- `templates/` for fillable loop files
- `reference-packs/` for standards to study
- `cases/` for proof surface
- `prompts/` for loop support
- `adapters/` for tool contracts
- `plugins/` for future package shapes

## Draft Changes

- Add and connect v0.1 spec files.
- Add `.mimesis` protocol files and fill them for this run.
- Add folder-level README files.
- Add adapter and plugin status documents.
- Update proof and contribution docs.
- Verify local links and status claims.

## Proof Needed

Fresh verification output for file existence, README section order, local Markdown links, issue form syntax, and git whitespace.

## Boundary Risks

- The fantasy-marketing language could imply adoption that does not exist.
- Plugin documents could be mistaken for shipped packages.
- Reference packs could be mistaken for permission to copy surface.
- Case notes could overstate improvement without evidence.
