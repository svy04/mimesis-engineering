# Release Check Order

Status: local release preflight order guard.

`npm run audit:release-order` checks the command order inside `npm run release:check`.
It keeps the preflight in a generate before audit shape so generated packets and report-producing checks exist before dependent validators read them.

## Required Order

- Adapter packets are generated before adapter audits. The Superpowers adapter is generated before its dedicated audit.
- Publication, proof, proof-readiness, proof intake check, proof redaction, proof submission, first proof candidate, license, plugin install, plugin release, publish, gate-board, operator-runbook, ecosystem-resource, claim, case-publication, evidence-from-case, benchmark, adoption packet, release-execution, release-decision, release-evidence, publication evidence packet, gap register, gap closure plan, gate evidence packet, owner action queue, owner decision intake, owner decision answer record, owner answer review, owner evidence bundle, owner evidence intake record, owner evidence review, owner evidence attachment form, owner proof handoff, owner proof input, owner proof input issue, owner proof input request, owner proof input issue convert, owner proof input split, owner evidence submission record, owner evidence submission check, current state summary, gate closure readiness, worktree review packet, release review bundle, goal completion audit, and release artifact manifest packets are generated before their dependent audits.

## Boundary

This is a local script-order audit.
It does not publish, stage, commit, push, tag, release, choose a license, create external proof, or prove remote freshness.
