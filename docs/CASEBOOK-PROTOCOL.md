# Casebook Protocol

Use this when turning a Mimesis run into a public case.

The protocol is informed by the neighboring `mimesis-casebook` repository.

## Minimum Case Shape

Every public case should show:

1. Starting Artifact
2. Mimesis Lens or Reference Structure
3. What Changed
4. What Remains Unproven
5. Next Proof Artifact

See [External Casebook Index](../cases/EXTERNAL-CASEBOOK.md) for public examples from the neighboring casebook repo.
Use [Permissioned External Case Packet](PERMISSIONED-CASE-PACKET.md) before claiming a user-submitted external case.
Use [Case Review Checklist](CASE-REVIEW-CHECKLIST.md) before publishing a public or redacted case.
Use [Case Publication Packet](CASE-PUBLICATION-PACKET.md) after `case:check` passes and before copying a case into a public casebook.

## Stronger Case Shape

When enough evidence exists, add:

- reference artifacts
- knowledge structure
- visual structure
- user flow
- before/after screenshots or text
- validation result
- reader/user feedback

## Case Types

- transformation case: an artifact changed
- boundary case: shallow copying was detected or repaired
- positioning note: a strategic direction changed, but outcomes remain unproven
- public-source specimen: public materials are used as reference standards without implying customer proof or endorsement

## Not Enough For A Case

- a vague idea
- a prompt with no before/after
- a claim without a boundary
- a transformation with hidden references
- a performance claim without evidence
- a permissioned case without review, permission, and redaction boundaries

## Boundary

A case proves that a transformation was documented.
It does not prove universal effectiveness, external adoption, or commercial success.
