# Originality Boundary

Mimesis is not permission to copy.
It is a discipline for extracting structure without stealing surface.

Weak Mimesis copies surface.
Strong Mimesis extracts structure.

| Do Not Copy | Extract Instead |
| --- | --- |
| exact sentences | argument structure |
| exact layout | information hierarchy |
| brand voice | trust devices |
| visual identity | interaction pattern |
| claims | claim boundary |
| social proof | proof mechanism |
| examples without attribution | reusable pattern with context |

## What Counts as Weak Mimesis

Weak Mimesis happens when the output mainly resembles the reference.

Warning signs:

- same wording rhythm
- same layout
- same claims
- same visual identity
- no new constraints
- no proof boundary
- no explanation of transformation

## What Counts as Strong Mimesis

Strong Mimesis happens when the output has its own constraints and a clear artifact trail.

It can explain:

- what was studied
- what structure was extracted
- what was transformed
- what was rejected
- what remains unproven

## Reference Etiquette

Keep a record of the references.
Do not hide the source of structure.

Use references as teachers, not costumes.

## When to Attribute

Attribute when:

- a specific artifact shaped the structure
- a reader would reasonably want the source
- the case note depends on the comparison
- the reference is central to the lesson

## When Not to Use a Reference

Do not use a reference when:

- the only useful thing is its wording
- the surface is too distinctive
- the claims depend on proof you do not have
- the output would confuse audiences about ownership

## Boundary Check Questions

- Did I copy wording?
- Did I copy layout too closely?
- Did I hide the reference?
- Did I borrow authority without evidence?
- Did I transform the structure under my own constraints?
- Can I explain what changed?
- Can I explain what remains unproven?

## Case Note Requirement

If the work is public, write a case note.

The case note should show:

- starting artifact
- reference artifacts
- extracted structure
- transformation
- before / after
- what improved
- what remains unproven

No proof, no claim.
No transformation, no Mimesis.
