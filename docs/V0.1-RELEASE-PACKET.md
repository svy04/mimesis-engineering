## Optional Workspace Preflight

If the neighboring repositories are checked out beside this one, run:

```bash
npm run release:check:workspace
```

This runs the required local preflight and then audits the local ecosystem shape across:

- `mimesis-engineering`
- `mimesis-canvas`
- `mimesis-casebook`

If public GitHub repository metadata can be read, run:

```bash
npm run release:check:public
```

This adds read-only remote visibility checks and a forced fallback audit for the expected public repositories.
It does not prove remote content freshness or external adoption.

To require a clean worktree synced with upstream before publication:

```bash
npm run release:ready:publish
```

This is expected to fail while there are uncommitted or untracked local changes.

## Publication Notes

When publishing v0.1, say:

```text
Mimesis Engineering v0.1 is a public working framework for giving AI standards, not roles. Bring one weak artifact, run the file protocol, and leave with a stronger artifact plus a proof boundary.
```

Do not say:

```text
This proves Mimesis improves productivity, adoption, customer outcomes, or legal originality.
```

## Stop Conditions

Do not publish a stronger claim if:

- `npm run release:check` fails
- `npm run audit:activation` fails or the README no longer preserves the first-action path
- `npm run audit:first-loop` fails or the first-loop demo is described as external proof
- `npm run audit:framework-manifest-schema` fails or the schema is described as external adoption, package publication, shipped plugin, official host compliance, customer outcome, or legal originality proof
