# Mimesis Engineering v0.1

## Status

Version: `v0.1`

Status: public working framework.

Claim boundary:
This is a usable method and file protocol.
It is not yet a benchmarked productivity system, legal originality guarantee, or industry standard.

## Identity

Mimesis Engineering is an artifact-first work system for AI-native builders.

Fantasy-marketing identity:
Mimesis Engineering is a field manual for the next class of AI-native builders.

## North Star

Give AI standards, not roles.

Korean:
AI에게 역할이 아니라 기준을 줘라.

## Enemy

Roleplay Prompting.

Roleplay prompting asks AI to pretend.
Mimesis gives AI something real to study.

## User Promise

Bring one weak artifact.
Leave with a stronger one.

## One Loop

One Mimesis loop must produce:

1. Artifact Brief
2. Reference Set
3. Structure Map
4. Transformation Plan
5. Improved Artifact
6. Boundary Check
7. Case Note

## Framework Flow

| Stage | Action | Output |
| --- | --- | --- |
| M | Map the field | Reference candidates |
| I | Identify strong artifacts | Reference Set |
| M | Model their structure | Structure Map |
| E | Extract patterns | Transformation Plan |
| S | Synthesize your version | Improved Artifact |
| I | Inspect the boundary | Boundary Check |
| S | Ship proof | Case Note |

## Boundary

Weak Mimesis copies surface.
Strong Mimesis extracts structure.

No proof, no claim.
No transformation, no Mimesis.

## v0.1 Complete Enough When

- a stranger understands it in 30 seconds
- a builder can try it in 5 minutes
- the file protocol is visible
- reference packs exist
- source provenance is visible
- at least one case note exists
- originality boundary is clear
- proof boundary is clear
- adapters have integration contracts
- contributors can submit weak artifacts, cases, reference packs, and boundary risks

## Repository Surface

The v0.1 framework surface is:

- `README.md` - 30-second explanation and 5-minute first loop
- `spec/` - framework contracts
- `.mimesis/` - current artifact trail and copyable protocol
- `templates/` - fillable files for one loop
- `reference-packs/` - standard fields to study
- `docs/SOURCE-FIRST-PROTOCOL.md` - source provenance and primary-reference discipline
- `cases/` - before/after proof surface
- `examples/` - starting scenarios
- `docs/ECOSYSTEM.md` - relationship between framework, canvas, and casebook
- `docs/CASEBOOK-PROTOCOL.md` - minimum public case shape
- `docs/EVIDENCE-PACKET.md` - strong-claim evidence boundary
- `prompts/` - prompt helpers for the loop
- `adapters/` - tool integration contracts
- `plugins/` - future plugin shapes with explicit status labels
- `tools/` - local protocol checks
- `STATUS.md` - current claim and implementation status
- `ROADMAP.md` - proof-gated future work
- `CHANGELOG.md` - change history
- `LICENSE.md` - license boundary
- `SECURITY.md` - security boundary
