# AI Workflow Reference Pack

## Use When

Use this when an AI workflow produces output but no durable artifact trail.

## Source Quality

Prefer official tool docs, source repos, protocol specs, papers, standards, and maintained examples with explicit inputs and outputs.
Avoid hidden prompts, private evaluations, and unverified automation claims.

## What Good Artifacts Usually Do

- define the input artifact
- state the role of AI
- name constraints
- include verification
- preserve memory
- produce an output artifact
- make failure visible

## What to Inspect

- input artifact
- role of AI
- constraints
- verification
- memory
- output artifact

## Transferable Patterns

- input-before-generation
- constraint block
- artifact checklist
- verification gate
- memory update
- proof note

## Do Not Copy

- private data flows
- proprietary prompts
- hidden evaluations
- unverified automation claims
- authority language without proof

## Starter Prompt

```text
Study the reference AI workflows below.
Extract their structure, not their wording.

Identify input artifact, AI role, constraints, verification, memory, output artifact, and failure handling.

Then transform my workflow so it leaves an artifact trail and proof boundary.
```
