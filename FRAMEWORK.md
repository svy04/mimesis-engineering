# Framework

The MIMESIS loop turns references into standards, then standards into a stronger artifact.

It is complete only when it leaves artifacts behind.

The default artifact trail lives in [`.mimesis/`](.mimesis/README.md).
The fillable files live in [`templates/`](templates/README.md).

## 1. Map the Field

List the products, papers, repos, creators, failures, and market structures that define the space.

Question:
What artifacts define this space?

Output:
Reference candidates.

Protocol file:
`.mimesis/reference-set.md`

## 2. Identify Strong Artifacts

Choose artifacts worth learning from because they are clear, trusted, widely used, deeply made, or unusually effective.

Question:
Which examples are worth studying?

Output:
Reference Set.

Protocol file:
`.mimesis/reference-set.md`

## 3. Model Their Structure

For each artifact, decompose:

- knowledge structure
- visual structure
- user flow
- trust device
- claim boundary
- failure handling
- rhythm and taste

Question:
What makes them work?

Output:
Structure Map.

Protocol file:
`.mimesis/structure-map.md`

## 4. Extract Transferable Patterns

Separate:

- patterns to reuse
- patterns to transform
- patterns to reject
- elements that would become plagiarism if copied

Question:
What can transfer without copying?

Output:
Transformation Plan.

Protocol file:
`.mimesis/transformation-plan.md`

## 5. Synthesize Your Version

Create your version under your actual constraints.

Question:
What should your artifact become?

Output:
Improved Artifact.

Protocol file:
`.mimesis/improved-artifact.md`

## 6. Inspect the Boundary

Ask where the work becomes:

- shallow imitation
- aesthetic copying
- unsupported authority
- hype
- unclear ownership
- weak proof

Question:
Where could this become shallow, copied, or overclaimed?

Output:
Boundary Check.

Protocol file:
`.mimesis/boundary-check.md`

## 7. Ship Proof

Choose evidence that decides the next move:

- user feedback
- benchmark
- test
- public critique
- case study
- proof artifact

Question:
What evidence or case note proves the change?

Output:
Case Note.

Protocol files:

- `.mimesis/case-note.md`
- `.mimesis/run_ledger.md`

## Local Validation

Run:

```bash
node tools/validate-mimesis.mjs
```

This checks local protocol consistency.
It does not prove external adoption, productivity gains, or legal originality.
