# Framework

## 1. Map the Field

List the products, papers, repos, creators, failures, and market structures that define the space.

## 2. Identify Predecessors

Choose artifacts that are worth learning from because they are clear, trusted, widely used, deeply made, or unusually effective.

## 3. Model Their Structure

For each artifact, decompose:

- knowledge structure
- visual structure
- user flow
- trust device
- claim boundary
- failure handling
- rhythm and taste

## 4. Extract Transferable Patterns

Separate:

- patterns to reuse
- patterns to transform
- patterns to reject
- elements that would become plagiarism if copied

## 5. Synthesize Your Variation

Create your version under your actual constraints.

## 6. Inspect Failure Modes

Ask where the work becomes:

- shallow imitation
- aesthetic copying
- unsupported authority
- hype
- unclear ownership
- weak proof

## 7. Ship, Test, Revise

Choose evidence that decides the next move:

- user feedback
- benchmark
- test
- public critique
- case study
- proof artifact
