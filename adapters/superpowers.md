# Superpowers Adapter

Status: `contract`

Adapter source: `adapters/superpowers.md`

Superpowers is a process-discipline layer for agent work.
Mimesis Engineering is the artifact-standard and proof-boundary layer.
This adapter shows how to use both without turning either one into roleplay.

## Contract

- Use Superpowers to choose and follow the right working discipline before action.
- Use Mimesis Engineering to define the artifact standard, reference boundary, output contract, and proof trail.
- Keep the loop visible: Import -> Distill -> Capsule -> Shard -> Verify -> Remember.
- Keep the two public standards visible:
  - Give AI standards, not roles.
  - Bring one weak artifact.
- Use tests, audits, or checklists before claiming completion.
- Update `.mimesis/run_ledger.md` only with evidence-backed results.

## Phase Map

| Mimesis phase | Superpowers discipline |
| --- | --- |
| Import | Use the relevant skill before acting; read repo/runtime truth. |
| Distill | Convert the request into standards, constraints, and proof boundaries. |
| Capsule | Split the work into one verifiable slice. |
| Shard | Implement by file/module while preserving the artifact trail. |
| Verify | Run the smallest meaningful audit first, then wider preflight. |
| Remember | Record what changed, what passed, and what gates remain open. |

## Boundary

This adapter does not install Superpowers.
It does not prove that a Superpowers skill ran.
It does not prove package publication, external adoption, benchmarked productivity, legal originality, shipped plugins, or real user outcomes.
It only gives agents a local contract for combining Superpowers process discipline with Mimesis artifact standards.
