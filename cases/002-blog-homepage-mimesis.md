# Case 002 — Blog Homepage Mimesis

Source:
Adapted from the neighboring `mimesis-casebook` case `002-blog-homepage-mimesis.md`.

This local version expands the case into the framework's proof-boundary format.

## 1. Starting Artifact

The starting artifact was a blog homepage for a personal methodology and proof surface.

The homepage had strong concepts, but the front page carried too many competing proof, stakeholder, and positioning sections.

## 2. Problem

The homepage risked making a first-time reader work too hard.

The weak pattern:

- too many proof sections too early
- too many stakeholder paths
- concept depth before first-screen clarity
- visual previews that did not fully support the reading path
- code and terminal blocks that needed clearer readability

## 3. Mimesis Lens

Good concept hubs reduce first-screen cognitive load.

They usually:

- expose one main thesis
- make the first path obvious
- move depth into reading paths
- show proof boundaries without drowning the first screen
- keep the source of truth separate from distribution surfaces

## 4. Reference Structure

Reference categories:

- concept hubs
- start-here pages
- proof-boundary pages
- case-study indexes
- technical blogs with readable code and terminal blocks

The case uses these as structure references, not as wording or layout sources.

## 5. What Changed

The local casebook record says the homepage changed by:

- removing performative section labels from the front
- keeping the plain thesis: "AI가 만든 결과를 그냥 믿지 않기 위해 씁니다."
- reducing the first path to method, case, boundary
- fixing preview image cropping
- improving terminal and code readability

## 6. Before / After

Before:
A homepage with strong concepts but too many simultaneous proof and stakeholder signals.

After:
A concept hub that routes the reader through:

- method
- case
- boundary

## 7. What Improved

- lower first-screen cognitive load
- clearer thesis
- cleaner route into deeper materials
- better proof-boundary placement
- better readability for technical snippets

## 8. What Remains Unproven

- external reader conversion
- audit demand
- long-term retention
- whether the simplified route improves real reader action

## 9. Next Proof Artifact

The next proof artifact should include:

- before/after screenshots
- reader feedback
- click or navigation evidence if available

## Boundary

This case is evidence of a documented homepage transformation.
It is not evidence of external adoption, commercial validation, or universal homepage effectiveness.
