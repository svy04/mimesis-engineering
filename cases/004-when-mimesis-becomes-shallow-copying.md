# Case 004 — When Mimesis Becomes Shallow Copying

## Failure Pattern

If the output only looks like the reference but does not transform the structure under its own constraints, it is not strong Mimesis.

## What Went Wrong

The builder studied a strong artifact and copied its surface.

The output had:

- similar layout
- similar wording rhythm
- similar claims
- no new constraints
- no proof boundary
- no transformation explanation

It looked better at a glance.
It was weaker as Mimesis.

## Warning Signs

- same layout
- same wording rhythm
- same claims
- no new constraints
- no proof boundary
- no transformation explanation

## How to Repair It

Return to the Structure Map.

Then:

1. name what was extracted
2. remove surface similarity
3. rebuild under your own audience
4. rebuild under your own constraints
5. rebuild under your own proof
6. rebuild toward your own goal

## Boundary Questions

- Did I copy wording?
- Did I copy layout too closely?
- Did I borrow authority without evidence?
- Did I hide the reference?
- Did I add my own constraints?
- Did I state what remains unproven?
- Can I explain the transformation?

## Better Version

Weak version:

```text
This looks like the reference.
```

Better version:

```text
This uses the reference's problem structure and proof rhythm,
but changes the audience, evidence, constraints, and claim boundary.
```

## Lesson

Weak Mimesis copies surface.
Strong Mimesis extracts structure.

No transformation, no Mimesis.
